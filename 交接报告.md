## 3. 架构快照

### 3.1 Kernel（纯规则层）

| 模块 | 职责 |
|------|------|
| `TaskStatus` DU | Pending/Running/Submitted/Merged/Done/Cancelled + `tryTransition` 穷举 |
| `Dag` | `Map<TaskId, Task>` + `sessionId` + `rootRequirement` |
| `topoSort` | DFS + visiting/visited 双集合，环检测 |
| `EventPayload` + `foldEvent/foldAll` | 七类事件 → DAG 纯折叠（SSOT 重放核心）|
| `gitReconcileDag` | **新增**：纯函数，git 二次校正 DAG（Cancelled 不覆盖）|
| `tick` | Scheduler 纯决策：给定 DAG+config → `toStart` + `events` |
| `buildSlavePrompt` | 初始 prompt 构造（有/无 万象术 两分支）|
| `buildEventProse` | 事件 → 用户友好正文（含 "Nothing needs to be done" 标注）|
| `SquadConfig` | `maxConcurrent/masterBranch/terminal/sharedDirs` |

### 3.2 Shell（副作用边界）

| 模块 | 职责 |
|------|------|
| `PromiseQueue` | `SerialQueue`（`tail` 可变 + `Promise.catch/bind` 锁队尾）|
| `PromiseCompat` | `fromTask`/`fromTaskUnit`（.NET Task → JS.Promise 桥接）|
| `GitExecutor` | `IGitExecutor`：`ResolveMasterBranch/FastForward/CreateWorktree/RemoveWorktree/IsAncestor` 等 7 操作 |
| `HttpServer` | Node `http.createServer` + Bearer token 鉴权 + 5 路由 + `Promise.catch` |
| `CoordinatorState` | 唯一可变 DAG 持有者；`ApplyDag/GitReconcile/RegisterPid/OnSubmitResult/OnSlaveExit` |
| `EventCodec` | YAML frontmatter 解析（全量 `yaml` 包）+ `decodeSquadEvent/decodeTaskId` |
| `ConfigReader` | AGENTS.md frontmatter `squad:` 节读取（`readSquadConfig`）|
| `PidMonitor` | `process.kill(pid,0)` 跨平台探活 + `System.Threading.Timer` 轮询 |
| `SlaveSpawn` | 终端命令映射（7 终端）+ `spawnSlave` + `createSymlinks` |
| `NodeInterop` | `currentPid`/`execGit`/`httpGet/httpPost`/`pathJoin`/`fs*Sync` |
| `SessionInject` | `injectQueue` 持有（注入序列化器）|

### 3.3 Plugin（宿主接线）

| 模式 | 入口 | 关键逻辑 |
|------|------|----------|
| Coordinator | `pluginForCoordinator` | HTTP server 启动 + `/squad` & `/squad-kill` command handler + `squad_update` tool + PID 轮询 + 重放 |
| Slave | `pluginForSlave` | `submit_to_squad` + `query_squad` + `POST /register` + `dispose` done beacon |
| 双模式探测 | `detectMode` | `SQUAD_COORDINATOR_URL` 环境变量区分 |

## 6. 关键路径表

| 路径 | 触发 | 处理 | 输出 |
|------|------|------|------|
| `/squad <requirement>` | `command.execute.before` | 注入 `squad_command=create` frontmatter → LLM 调 `squad_update` | 拆解 prompt + 等待 LLM 调用工具 |
| `squad_update` tool | LLM 调工具 | `parseEvents` → 拓扑校验 → `ApplyDag` → `tick` → `spawnSlave` | `task_created` + `task_started` 事件注入 |
| `POST /submit` | slave `submit_to_squad` | `SerialQueue` → `FastForward`（前置校验 + `merge-base` + `merge --ff-only`）| `merged` / `rebase_needed` / `stale_commit` 等 5 分支 |
| `POST /register` | slave 启动 | `CoordinatorState.RegisterPid` | `registered` |
| `POST /done` | slave `dispose` | `CoordinatorState.OnSlaveExit` → `removeWorktree` → `Scheduler.tick` | `acknowledged` |
| PID 轮询（2s）| 定时器 | `PidMonitor.alive` → `OnSlaveExit` | `task_done` 事件注入 + `removeWorktree` |
| `/squad-kill` | `command.execute.before` | `applyDag [squad_cancelled]` + `process.kill(pid, SIGTERM)` | 保留 worktree，`cancelled` 终态 |
| 重放 | `tryReplayOnce` | `session.messages()` → `parseFrontMatter` → `ReplayFromHistory` → `GitReconcile` | DAG 内存态重建 |
